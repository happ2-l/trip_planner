# 도쿄 · 東京 (설치형 PWA)

친구 3명과 함께 쓰는 도쿄 여행 앱. 와시(washi) 감성의 클래식 일본풍 디자인.
- 🗓️ **일정**: 3일 타임라인 · 시간/제목 탭하면 수정 · 완료 체크 · 메모 · 직접 추가/삭제
- 🗺️ **지도**: 실제 OpenStreetMap(무료) + 번호 핀, 탭하면 장소 상세
- 📍 **장소**: 명소/맛집 · 상세 시트(영업시간·팁) · **1인당 "가고싶어요" 투표**
- 🧮 **정산**: 총지출/1인당 자동 계산 · 정산 방법(누가 누구에게) · 지출 추가/수정
- 🎒 **준비**: 그룹 체크리스트(장마 대비 포함) + 항공/숙소/교통/긴급 정보
- 📲 홈 화면에 설치(아이콘 생성), 오프라인 실행
- 🔄 친구들끼리 **실시간 공유** (Firebase 무료) — 일정·체크·투표·정산 모두 동기화
- 🙋 친구 이름은 **정산 탭에서 이름을 탭**해 수정 (기본: 민준·서연·지호)

---

## 1) 빠른 시작 (그냥 써보기)

설정 없이도 바로 동작합니다. 다만 이 경우 데이터는 **각자 기기에만** 저장되고 공유되지 않습니다.
실시간 공유를 원하면 **3번**을 진행하세요.

---

## 2) 무료 호스팅에 올리기 (폰에 "설치"하려면 필수)

PWA가 홈 화면에 설치되려면 `https://` 주소가 필요합니다. 아래 중 **택1**.

### A. GitHub Pages (무료)
1. https://github.com 에서 새 저장소(repository) 생성 (예: `tokyo-trip`).
2. 이 폴더의 **모든 파일**을 업로드 (드래그&드롭 가능).
3. 저장소 → **Settings → Pages** → Branch를 `main` / 폴더 `/ (root)` 로 지정 → Save.
4. 잠시 뒤 `https://<아이디>.github.io/tokyo-trip/` 주소가 생깁니다. 이 링크를 친구들에게 공유!

### B. Netlify (더 쉬움, 드래그&드롭)
1. https://app.netlify.com → **Add new site → Deploy manually**.
2. 이 폴더를 통째로 드래그&드롭 → 끝. 생성된 주소를 공유.

### C. 폰에 설치하는 법 (친구들에게 안내)
- **아이폰(Safari)**: 링크 열기 → 공유 버튼 → **홈 화면에 추가**
- **안드로이드(Chrome)**: 링크 열기 → 우상단 ⋮ → **앱 설치 / 홈 화면에 추가**

---

## 3) 실시간 공유 설정 (Firebase · 5분, 무료)

친구가 일정을 체크/메모하면 모두의 폰에 바로 반영됩니다.

1. https://console.firebase.google.com → **프로젝트 추가** (이름 아무거나).
2. 좌측 메뉴 **빌드 → Realtime Database → 데이터베이스 만들기**
   - 위치: `asia-southeast1` 등 아무거나
   - 보안 규칙: **테스트 모드로 시작** 선택 (또는 아래 규칙 붙여넣기)
3. 프로젝트 설정(⚙️) → **내 앱 → 웹 앱 추가(</>)** → 앱 등록.
   표시되는 `firebaseConfig` 값(apiKey, databaseURL 등)을 복사.
4. 이 폴더의 **`firebase-config.js`** 를 열어 값들을 붙여넣기. (그 후 다시 업로드)
5. 세 명 모두 **같은 주소**로 접속하면 자동으로 같은 방에서 공유됩니다.
   (`TRIP_ID` 값이 같으면 같은 방 — 기본값 그대로 두면 됨)

### 보안 규칙 (테스트용, 우리끼리만 쓸 때)
Realtime Database → **규칙** 탭에 붙여넣기:
```json
{
  "rules": { ".read": true, ".write": true }
}
```
> ⚠️ 누구나 읽고 쓸 수 있는 규칙입니다. 여행 끝나면 Database를 삭제하거나
> 규칙을 `false`로 바꿔 잠그는 걸 권장합니다. (주소를 모르면 접근은 어렵습니다.)

연결 상태는 앱 우상단에 표시됩니다:
- `● 실시간 연결됨` → 공유 ON
- `● 이 기기에 저장` → 설정 전(로컬 모드)

## 일정 / 맛집 내용 바꾸기
- 일정·맛집·준비물·정보 텍스트는 **`data.js`** 한 파일에서 수정합니다.
- 앱 안에서 일정/준비물을 직접 추가·삭제할 수도 있습니다(공유됨).

## 아이콘 다시 만들기 (선택)
```
python gen_icons.py
```

## 파일 구조
```
index.html            화면
styles.css            디자인
data.js               일정·맛집·정보 데이터 (여기 수정)
app.js                앱 로직 + 실시간 동기화
firebase-config.js    공유 설정 (값 붙여넣기)
manifest.webmanifest  PWA 설치 정보
sw.js                 오프라인 캐싱
icons/                앱 아이콘
gen_icons.py          아이콘 생성기
```
